# Daily Briefing 前端界面设计

> 用于展示 `daily_news/*.md` 的阅读端。先定稿界面与交互，再开发。参考 00 计划 §2.2：Notion 风、简洁科技感、真实页面设计，避免「明显 Markdown 布局」。

---

## 1. 产品目标与用户

- **目标**：用户打开前端，能按日期浏览每日 AI 简报，单篇可从头到尾舒适读完（10～15 分钟）。
- **用户**：关注 AI 的工程师、产品/设计，对阅读体验有要求，不希望像「渲染后的原始 Markdown」。
- **数据**：来自 `daily_news/YYYY-MM-DD_daily_news.md`，frontmatter（title, date, description, slug）+ 正文（摘要行已抽到 description，正文为三章节 Markdown）。

---

## 2. 页面结构（两页即可）

| 页面 | 路径示例 | 作用 |
|------|----------|------|
| **列表页** | `/` 或 `/daily` | 展示所有已有日报，按日期倒序，每条为一张卡片/一行，点进详情。 |
| **详情页** | `/daily/2026-03-17` 或 `/daily/daily-2026-03-17` | 展示单日完整简报，顶部日期+标题+摘要，下为三章节正文。 |

无需登录、无需后台；只读展示。

---

## 3. 列表页设计

### 3.1 布局示意（文字线框）

```
+--------------------------------------------------+
|  今日 AI 简报                    Daily Briefing|
+--------------------------------------------------+
|                                                  |
|  今日 AI 简报，少而精、可读完。                      |
|  按日期倒序，点击进入当日完整内容。                   |
|                                                  |
+--------------------------------------------------+
|                                                  |
|  +--------------------------------------------+  |
|  | 2026-03-17                                 |  |
|  | Meta 收购 Moltbook 加速 Agent 生态布局…    |  |
|  | 阅读全文 →                                  |  |
|  +--------------------------------------------+  |
|                                                  |
|  +--------------------------------------------+  |
|  | 2026-03-16                                 |  |
|  | 摘要首句或 description 前 80 字…            |  |
|  | 阅读全文 →                                  |  |
|  +--------------------------------------------+  |
|                                                  |
|  … 更多日期 …                                    |
|                                                  |
+--------------------------------------------------+
|  数据来自 daily_news，每日自动更新。                |
+--------------------------------------------------+
```

### 3.2 列表页要点

- **头部**：站点名（如「AI 日报」或「Daily Briefing」），可加一句简短 Slogan；右侧可预留暗色模式切换。
- **卡片**：
  - 主信息：**日期**（突出，如 2026-03-17）、**摘要**（description，1～2 行截断，如 80～100 字）。不展示「AI 日报 YYYY-MM-DD」标题行，避免与日期重复。
  - 操作：整卡或「阅读全文」可点击，进入对应详情页（用 slug 或 date 做路由）。
- **顺序**：按日期**倒序**（最新在上）。
- **风格**：留白充足、卡片有轻微分隔或阴影，不要密密麻麻列表感。

## 4. 详情页设计

### 4.1 布局示意（文字线框）

```
+--------------------------------------------------+
|  [Logo]  AI 日报              [返回列表]  (暗色)   |
+--------------------------------------------------+
|                                                  |
|  2026-03-17                                      |  ← 日期（小号、灰色或次要色）
|  AI 日报 2026-03-17                              |  ← 标题（大号、字重突出）
|                                                  |
|  Meta 收购 AI 社交网络 Moltbook 加速 Agent 生态  |
|  布局，ServiceNow 企业级 AI 代理功能渗透率达 40%  |
|  …（摘要/description，作为 lead 段落）             |
|                                                  |
+--------------------------------------------------+
|                                                  |
|  本周值得关注的 AI 产品与工具                      |  ← 章节标题（清晰层级，非「##」感）
|  ─────────────────────────────────────────────  |
|                                                  |
|  Moltbook：Meta 收购的 AI 代理社交平台             |  ← 条目标题（### 对应）
|                                                  |
|  背景：Moltbook 是专为 AI 代理设计的社交网络…     |
|  要点：此次收购金额未披露…                         |
|  使用人群 / 为何值得关注：多智能体系统开发者…      |
|                                                  |
|  [原文链接]（来源：NBC News）                     |  ← 链接区（可按钮化或下划线）
|                                                  |
|  ─────────────────────────────────────────────  |
|  ServiceNow 苏黎世版本（Zurich Release）          |
|  …                                               |
|                                                  |
+--------------------------------------------------+
|  各场景下的头部模型与玩家                          |
|  ─────────────────────────────────────────────  |
|  … 同上：场景名 + 若干代表模型/厂商 + 链接 …       |
+--------------------------------------------------+
|  今日 AI 大事件与重要言论                         |
|  ─────────────────────────────────────────────  |
|  … 事件名 + 发生了什么 + 为何值得关注 + 链接 …     |
+--------------------------------------------------+
|                                                  |
|  [返回列表]                                       |
+--------------------------------------------------+
```

### 4.2 详情页要点

- **顶部区（Hero / 文章头）**
  - **日期**：小号、次要色，例如 `2026-03-17`。
  - **标题**：大号、字重突出（如 1.25rem～1.5rem），与 00 计划「真实页面设计」一致。
  - **摘要**：description 作为 lead 段落，字号略大于正文或字重略轻，与正文有区分。
- **正文区**
  - **章节**：三个大块对应「本周值得关注的 AI 产品与工具」「各场景下的头部模型与玩家」「今日 AI 大事件与重要言论」。章节标题用**字体层级 + 间距**区分，避免「赤裸的 ##」感（例如更大字号、更多上边距、可选细线或色块分隔）。
  - **条目**：每个 ### 对应一个「条目块」。块内：
    - 条目标题（产品名/场景名/事件名）单独一行，字重或字号略大于正文。
    - 「背景」「要点」「使用人群 / 为何值得关注」「发生了什么」「为何值得关注」等**标签**可用小号、灰色或加粗，与正文区分。
    - 段落之间留白清晰，不要挤在一起。
  - **链接**：`[原文链接](URL)（来源：XXX）` 可渲染为按钮或带下划线的链接 + 灰色「来源：XXX」，便于点击且不抢眼。
- **底部**：可放「返回列表」链接，或简短版权/数据说明。

## 5. 视觉与风格（原则级）

| 维度 | 要求 |
|------|------|
| **气质** | Notion / The Verge / MIT Technology Review：简洁、科技感、不花哨。 |
| **字体** | 层级清晰：日期 < 摘要 < 标题 < 章节标题 < 条目标题；正文字号与行高保证可读（如 16px+ / 1.6 行高）。 |
| **留白** | 章节之间、条目之间、段落之间留白充足，避免「堆砌感」。 |
| **色彩** | 主色可极简（黑/深灰 + 白/浅灰），链接与标签用同一强调色（如蓝或品牌色）；可选支持暗色模式。 |
| **区分 Markdown** | 不依赖「# / ## / ###」的默认渲染样式，用**组件 + class** 做章节、条目、标签的差异化，使页面像 editorial 文章页。 |

---

## 6. 响应式与技术假设

- **响应式**：列表页卡片在窄屏下单列、宽屏可考虑两列；详情页正文最大宽度限制（如 720px～800px），避免长行影响阅读。
- **数据来源**：前端可假设有一份「日报列表」数据（日期、slug、title、description）和「单日报正文」（Markdown 或已解析的 HTML/JSON）。实现时可以是：构建时读 `daily_news/*.md` 生成静态页（SSG），或本地/服务器提供简单 API 返回列表与单篇内容。
- **技术栈**：不在此文档绑定；确认界面后可用 React + 静态站点（如 Next.js SSG / Vite + 静态导出）或其它方案实现。

## 7. 确认清单（请你过目）

- [ ] **列表页**：日期 + 标题 + 摘要预览 + 进入详情，倒序，卡片式；是否需增加「搜索/筛选」？（建议首版不做）
- [ ] **详情页**：日期 + 标题 + 摘要 + 三章节正文，章节与条目层级清晰、链接与来源明确；是否需「上一篇/下一篇」？（建议首版可做）
- [ ] **风格**：Notion/科技媒体风、留白充足、弱化「Markdown 感」；是否需要暗色模式？（建议首版支持切换）
- [ ] **其它**：是否需要首页「今日」快捷入口（自动跳转到最新一篇）？是否需要 RSS/订阅提示？

你确认或修改以上点后，再按此文档开始写前端代码。
